你是资深 Vue3 + TypeScript 前端工程师和 UI 设计系统专家。请基于当前项目实际结构优化聊天室前端，目标是"更好看、可维护、可上线"，并避免过重 AI 风。

【项目上下文（必须遵守）】
- 技术栈：Vue 3 + TypeScript + Vite。
- 路由页面：`/login` -> `client/src/views/Login.vue`，`/chat` -> `client/src/views/Chat.vue`。
- 聊天模块：`client/src/components/chat/ChatMessageList.vue`、`client/src/components/chat/ChatComposer.vue`、`client/src/components/chat/MembersModal.vue`。
- 数据与交互：保留现有 API 调用、WebSocket 收发、登录鉴权、会话切换、未读计数、图片消息、回复消息能力。
- 禁止破坏：接口字段、业务流程、路由行为、登录状态判断逻辑。

【视觉目标】
- 整体气质：明亮、温暖、年轻化、社交产品感。
- 风格关键词：毛玻璃、柔和渐变、轻阴影、大圆角、清晰层级。
- 避免：赛博朋克、重霓虹、夸张粒子、过度动画、深色压抑风。
- 配色建议：珊瑚粉、薄荷绿、橙色、青蓝色为主，控制饱和度，保证可读性。

【页面级要求】
1) 登录页（`Login.vue`）
- 中央登录卡片做玻璃拟态：半透明底 + 细边框高光 + 背景模糊。
- 背景做低频渐变和轻装饰光斑，不喧宾夺主。
- 输入框 focus 状态增强（边框/发光），按钮保留高对比可点击性。
- 登录/注册切换状态的视觉层级统一，错误提示不刺眼但醒目。

2) 聊天页主布局（`Chat.vue`）
- 左侧会话栏卡片化，当前会话高亮更明确，未读红点更清晰。
- 右侧聊天区分层：顶部信息区、消息滚动区、底部输入区。
- 消息区域留白更均衡，移动端和桌面端断点统一。
- 模态框（成员列表、用户信息）统一玻璃风格和阴影体系。

3) 消息与输入（`ChatMessageList.vue` + `ChatComposer.vue`）
- 自己消息：明亮渐变气泡；他人消息：浅色半透明气泡；文字对比达标。
- 图片消息边框/圆角/背景统一，点击预览反馈明确。
- 回复条、时间戳、昵称层级清晰；hover 仅做轻反馈。
- 输入区按钮（表情/图片/发送）保持一致尺寸、圆角、焦点态。

【头像清晰度优化（重点）】
- 头像容器固定尺寸，使用 `object-fit: cover`，避免拉伸和抖动。
- 使用高分屏策略：优先请求 2x 尺寸头像（如 OSS 参数 `?x-oss-process=image/resize,w_128,h_128` 等）。
- 对头像增加 1px 半透明描边和轻阴影，保证在亮背景下边界清晰。
- 列表滚动时避免头像重绘抖动：限制容器尺寸，避免频繁布局变化。

【代码工程化要求】
- 分层解耦：页面容器（view）负责状态编排，子组件负责展示与交互。
- 提取设计令牌：颜色、圆角、阴影、模糊、间距、动效时长集中管理（CSS 变量或 token 文件）。
- 删除重复样式与冗余选择器，降低 `!important` 使用。
- 保持代码可读性：必要注释只解释复杂逻辑，不写废话注释。

【动效与可访问性】
- 动效时长建议 120-240ms，使用 ease-out；仅在 hover/focus/切换时轻量触发。
- 保证键盘可达：按钮和输入框有可见 focus。
- 文本和背景对比度满足可读性，错误信息和禁用态可区分。

【输出格式（严格执行）】
- 先给"改造方案"（按文件列出）。
- 再给"可直接替换的代码"（按文件分块）。
- 最后给"验收清单"：
  - 视觉一致性
  - 交互可用性
  - 移动端适配
  - 头像清晰度
  - 性能与可维护性

【额外约束】
- 不引入大型 UI 框架。
- 优先原生 CSS + 现有组件结构。
- 只做前端界面与结构优化，不改后端接口语义。
- 输出内容必须能在当前项目中落地，而不是概念稿。
